# Queue (큐)

자료구조 queue에 대해서 학습하는 공간입니다

## 개요

큐(Queue)는 데이터를 줄을 서듯이 순서대로 처리하는 자료구조입니다. 가장 먼저 들어온 데이터가 가장 먼저 나가는 FIFO(First In First Out) 구조를 따릅니다.

## 주요 특징

- FIFO(First In First Out): 선입선출 방식
- 한쪽 끝(rear)에서 데이터 삽입, 다른 쪽 끝(front)에서 데이터 삭제
- 실생활 예시: 은행 대기줄, 프린터 출력 대기열, 놀이공원 줄서기

## 주요 연산

### enqueue
- 큐의 맨 뒤(rear)에 새로운 요소를 추가
- 시간 복잡도: O(1)

### dequeue
- 큐의 맨 앞(front)에 있는 요소를 제거하고 반환
- 시간 복잡도: O(1)

### peek (또는 front)
- 큐의 맨 앞(front)에 있는 요소를 제거하지 않고 조회
- 시간 복잡도: O(1)

### isEmpty
- 큐가 비어있는지 확인
- 시간 복잡도: O(1)

## 구현 방법

큐는 다음과 같은 방법으로 구현할 수 있습니다:

1. 배열(Array) 기반 구현
   - 선형 큐: 배열의 인덱스를 이용
   - 원형 큐(Circular Queue): 배열을 원형으로 사용하여 공간 효율성 향상
2. 연결 리스트(Linked List) 기반 구현
3. 덱(Deque): 양쪽 끝에서 삽입/삭제가 가능한 구조

## 큐의 종류

### 일반 큐 (Linear Queue)
- 기본적인 FIFO 구조의 큐

### 원형 큐 (Circular Queue)
- 배열의 마지막과 처음이 연결된 형태
- 메모리 공간 효율적 사용

### 우선순위 큐 (Priority Queue)
- 각 요소가 우선순위를 가짐
- 우선순위가 높은 요소가 먼저 처리됨
- 힙(Heap) 자료구조로 구현

### 덱 (Deque, Double-ended Queue)
- 양쪽 끝에서 삽입과 삭제가 모두 가능
- 스택과 큐의 기능을 모두 수행 가능

## 활용 사례

실제 애플리케이션 개발에서 큐를 사용할 수 있는 구체적인 사례들입니다.

### 프린터 작업 관리
**앱 예시**: Windows 인쇄 스풀러, macOS 프린트 센터
- **기능**: 여러 문서의 인쇄 요청을 순서대로 처리
- **구현**:
  - 인쇄 요청이 들어오면 큐에 enqueue
  - 프린터가 준비되면 큐에서 dequeue하여 인쇄 작업 수행
  - 대기 중인 작업 목록 표시
- **사용 시점**: 순차적 작업 처리가 필요한 모든 시스템

### 메시지 큐 시스템
**앱 예시**: 카카오톡, 슬랙, 디스코드
- **기능**: 메시지를 순서대로 전송하고 수신
- **구현**:
  - 발신자가 보낸 메시지를 큐에 저장
  - 서버가 순서대로 처리하여 수신자에게 전달
  - 네트워크 지연 시 큐에 임시 저장
- **사용 시점**: 채팅 앱, 이메일 시스템, 알림 서비스

### 고객 서비스 - 대기열 관리
**앱 예시**: 은행 앱, 병원 예약 시스템, 콜센터
- **기능**: 고객 대기 순서 관리
- **구현**:
  - 고객이 접수하면 큐에 enqueue
  - 상담사가 준비되면 큐에서 dequeue하여 다음 고객 호출
  - 현재 대기 인원 및 예상 대기 시간 표시
- **사용 시점**: 순번 대기 시스템, 티켓팅 시스템

### 동영상 스트리밍 - 버퍼링
**앱 예시**: YouTube, Netflix, Twitch
- **기능**: 네트워크에서 받은 영상 데이터를 순서대로 재생
- **구현**:
  - 네트워크로부터 받은 영상 청크를 큐에 enqueue
  - 재생기가 큐에서 dequeue하여 순서대로 재생
  - 버퍼링 상태 관리
- **사용 시점**: 미디어 플레이어, 오디오 스트리밍

### CPU 작업 스케줄링
**앱 예시**: 운영체제 프로세스 관리
- **기능**: 여러 프로세스를 공정하게 실행
- **구현**:
  - Ready Queue에 실행 대기 중인 프로세스를 저장
  - CPU가 프로세스를 순서대로 처리 (Round-Robin 스케줄링)
  - 시간 할당량만큼 실행 후 다시 큐의 뒤로 이동
- **사용 시점**: 멀티태스킹 시스템, 작업 스케줄러

### 게임 - 턴제 시스템
**앱 예시**: 리그 오브 레전드 매칭, 온라인 보드게임
- **기능**: 플레이어들의 턴 순서 관리
- **구현**:
  - 플레이어를 큐에 순서대로 저장
  - 현재 플레이어가 턴을 마치면 dequeue하고 다시 enqueue
  - 게임 매칭 대기열 관리
- **사용 시점**: 턴제 게임, 매칭 시스템

### 웹 서버 - 요청 처리
**앱 예시**: Nginx, Apache, Node.js 서버
- **기능**: 클라이언트 요청을 순서대로 처리
- **구현**:
  - HTTP 요청을 큐에 저장
  - 워커 스레드가 큐에서 요청을 가져와 처리
  - 요청 처리 순서 보장
- **사용 시점**: API 서버, 백엔드 시스템

### 백그라운드 작업 처리
**앱 예시**: Gmail(이메일 발송), Instagram(이미지 처리)
- **기능**: 시간이 오래 걸리는 작업을 백그라운드에서 순차 처리
- **구현**:
  - 이메일 발송, 이미지 리사이징 등의 작업을 큐에 추가
  - 백그라운드 워커가 큐에서 작업을 가져와 처리
  - 작업 완료 시 사용자에게 알림
- **사용 시점**: 비동기 작업 처리, 배치 작업

### 알고리즘 - 너비 우선 탐색 (BFS)
**앱 예시**: 최단 경로 찾기, 소셜 네트워크 친구 추천
- **기능**: 그래프나 트리를 너비 우선으로 탐색
- **구현**:
  - 시작 노드를 큐에 enqueue
  - 큐가 빌 때까지 dequeue하며 인접 노드들을 enqueue
  - 레벨 순서로 탐색하여 최단 경로 보장
- **사용 시점**: 최단 경로 찾기, 레벨 순서 순회

### 알림 시스템
**앱 예시**: Facebook, Instagram, LinkedIn 알림
- **기능**: 알림을 시간 순서대로 표시
- **구현**:
  - 새로운 알림이 발생하면 큐에 enqueue
  - 사용자가 앱을 열면 큐에서 순서대로 알림 표시
  - 오래된 알림부터 처리
- **사용 시점**: 푸시 알림, 실시간 업데이트
